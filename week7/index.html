Week 7 Notes

Ch 11: Further Functions

Immediately Invoked Function Expressions (IFFE or “iffy”)

  • Anonymous function that’s invoked as soon as it’s defined
  • Good for keeping variables out of scope (though not needed after ES6)
  • Good for creating temporary “run once” code
  • Can also be used to contain strict mode code

Functions that define and rewrite themselves

  • Redefine a function by assigning an anonymous function to a variable with the same name as the function.

Recursive Functions

  • Function that invokes itself until a certain condition is met

Callbacks

  • Callbacks are functions that are passed arguments to other functions then invoked inside the function it was passed to.
  • Callbacks make event driven asynchronous programming possible
    • JS is single threaded (one piece of code at a time)
    • Callbacks can be invoked when an event happens allowing the3 code to run asynchronously (out of order)
  • Avoid using multiple callbacks in a single function

Promises

  • Represents the future result of an asynchronous operation
  • Help avoid messy code from using multiple callbacks
  • Process
    • Promise is made - ordered a burger ands you get a tray and a buzzer as a promise that you will get a burger
    • The operation is pending - waiting for you r burger
      • The operation is said to be “unsettled”
    • Operation is completed (settled) - the buzzer goes off and your order is ready
    • Result is 2 different outcomes
      • Resolved - completed successfully - you pick up your delicious burger
      • Rejected - didn’t work or resulted in an error - there was something wrong with your burger or you decided not to wait any more and left.
  • The anatomy of a promise
    • const promise = new Promise( (resolve, reject) => { 

// initialization code goes here 

if (success) { resolve(value); 

} else { 

    reject(error); } });

Async Functions

  • Use the async keyword and let you write asynchronous code as if it was synchronous
    • Use the await operator before the asynchronous function
  • Process
    • Add the async keyword before the asynchronous function
    • Wrap each step of the function in a try block to catch errors
    • The await operator is added to make sure the next line of code isn’t executed until the current one is finished.

Closures

  • A reference to a variable that was created inside the scope of another function but is kept alive and used in another part of the program
    • Formed when an inner function is returned by an outer function allowing it to maintain access to any inner variables inside the inner function.

Generators

  • Special functions used to produce iterators that maintain the state of a value
  • To define 
    • function* exampleGenerator() { // code for the generator goes here } 
  • Doesn’t run code in the function - returns a generator object used to create an iterator
    • Iterator implements next() method that returns a value every time it’s called
    • Uses the yield keyword to return a value
      • Value is stored for use later unlike return
  • Good for creating sequences

Functional Programming

  • A programming paradigm characterized by the use of pure functions
    • Pure function rules:
      • Pure function return value comes from provided arguments not values found elsewhere in the program
      • No side-effects: pure functions don’t change va;ues or data anywhere else in the program - only non-destructive data transformations
        • Using const helps prevent destructive transformations too
      • Referential transparency: a function will always return the same results with the same arguments
    • Pure functions must have:
      • At least 1 argument
      • A return value

Currying

  • A process involving partially applying functions
    • Curried functions don’t have all arguments supplied so it returns another function that holds the provided arguments and waits for the rest of the arguments.
    • Result is returned when the expected arguments are finally provided.

Ch 13: Ajax

Ajax

  • A technique for allowing web pages to communicate with a server asynchronously and update a page without reloading.
  • Ajax allows Javascript to request resources from a server
    • Usually as JSON data or bits of text or HTML
  • Stands for Asynchronous JavaScript and XML

The Fetch API

  • Replaced XMLHttpRequest object
  • Provides a global fetch() method
    • This method takes one url as an argument and works as a promise
    • Example:

fetch('https://example.com/data') 

.then( // code that handles the response )

.catch( // code that runs if the server returns an error ) 

Response Interface

  • Processes the objects returned by a Fetch API promise with its own methods and properties

Form Data interface

  • Makes it easier to submit information using Ajax
  • Create a FormData instance (const data = new FormData(); ) to reduce the amount of code you need to write.
