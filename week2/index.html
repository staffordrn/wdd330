Chapter 2: Programming Basics

  • Comments
    • Help explain whats going on
    • “Hallmark of a ninja programmer”
    • // - single line comments
    • /* multiple line comment */
  • JavaScript Grammar
    • JS syntax = C style syntax due to similarities to C language
    • JS doesn’t require ; at end of line but do it anyway
    • Best Practice: statements start on a new line and end with ;
  • Reserved words
    • Can’t be used as variables, object properties, or function parameters
    • Undefined, NaN, and Infinity are not reserved words but should be
  • Primitive data types
    • String
    • Symbol
    • Number
    • Boolean
    • Undefined
    • Null
  • Variables
    • Must be declared before they can be used
    • Refer to a value stored in memory
    • Use const if the value should not change
    • Use let if value could be reassigned
    • Use assignment operator (=) to assign value
    • Assign multiple values at once - separate with commas
    • Making an object a constant does not prevent the data within the object from changing
    • What happened to ‘var’?
      • Let & const have block scope, var does not*
      • Let & const help prevent overwriting built in methods
      • Var can still be used but don’t
      • * block scope = variable only exists in the block it is declared in
    • Scope
      • Declared outside block = global scope
      • Declared in a block = only available to the block
      • Keep global variables to a minimum
  • Naming Constants and variables
    • Sensible, meaningful names
    • Start with lowercase letter, _, or $
    • Variable names are case sensitive
    • Use camelCase
  • Direct Assignment and Assignment by Reference
    • Direct: a = 1, b = 2…
    • By reference: changes made to an object affect all variables that refer to the object.
  • Strings
  • Create using single quotes (‘’), double quotes (“”), or backtick (`)
    • Use escape character to allow apostrophe, quotes, etc
  • String Properties and methods
    • Properties = info about object or value
    • Methods = perform action on object or value
    • Dot and bracket notation for methods: 
      • data.method() - dot
      • data [‘property’] - bracket
    • Concatenation: 
      • concat()
      • + operator
  • Template literals - special strings created using the backtick character
    •  don't need escape for (‘) or (“) 
    • Allows for JS code interpolation
      • Uses ${value}
    •  ex const name = ‘Bob’; `Hello ${ name}, it’s me`;
  • Symbols = new primitive value
    • Created using Symbol() function
    • For creating unique values
    • Add description inside ()
    • Mainly used for object property keys
  • Numbers
    • Integers for floats
  • Number Methods
    • Dot notation
      • 2 dots - 5..toExponential();
      • Space before dot - 5 .toExponential();
  • Changing variable values
    • a = a + 5 or a += 5
    • Incrementing  ++, --
    • All the usual arithmetic operations can be done in JS
  • Infinity and NaN
    • Infinity - special error for numbers too big or small for JS
    • NaN - result when operation returns non-numerical result
  • Type Coercion
    • JS will try to force non-matching types to match
      • Often produces errors
    • Best practice - be as explicit as possible about data type
  • Converting between string and number
    • Number() = string→ number
    • String() = number → string
    • toString() = number → string
    • parseInt() = string→ number
    • parseFloat = string→ decimal
  • Undefined = variable not assigned a value
  • Null = no value (acts like 0 in operations)
  • Booleans = true or false
  • Logical operators
    • ! = not
    • && = and
    • || =  or
  • Equality operators
    • == soft equality (no date type check)
    • === hard equality (checks data type)



Chapter 3: Arrays, Logic, and Loops

 

  • Array - an ordered list of values
    • To create empty array - const heroes = [];
    • To create and populate an array - const avengers = ['Captain America', 'Iron Man', 'Thor', 'Hulk']; (array literal)
    • Assign single values - use index: avengers[2] = 'Ant Man';
    • Items in arrays can be treated like variables
    • Delete array value - delete avengers[3];
    • Arrays can hold values of different data types
    • Array Properties and Methods
      • array.length() - returns array length and can also define array length
      • array.pop() - removes the last item from the array
      • array.shift() - removes the first item from the array
      • array.push() - appends a value to the end
      • array.unshift() - appends a value to the beginning
    • Merging Arrays
      • concat() merges two arrays but does not change the original arrays
      • myArray = myArray.concat([‘array’, ‘values’]) changes the original array and adds the new values
      • The spread operator can merge arrays too - myArray = [...myArray, ...[‘array’, ‘values’]
      • join() - turns the array into a comma separated string
      • slice() - creates a sub-array out of a section of an array
        • Doesn’t remove anything out of the original array
      • splice() - removes items and inserts their items in their place
      • reverse() - reverses the array order
      • sort() - sorts the items alphabetically (numerical character strings too)
      • indexOf() - returns the index of an item
      • includes() - returns a boolean (t/f) if an item is in the array
    • Multidimensional arrays = arrays of arrays
      • Good for coordinates
      • Spread operator is used to flatten multidimensional arrays - essentially combines all arrays so the nested levels are all in one array
  • Set - a list of unique values (no duplicates). Currently no literal notation
    • Create and empty set - const list = new Set();
    • Adding values to sets - use the add() method
      • list.add(2).add(3).add(4);
      • Add at creation - const numbers = new Set([1,2,3]);
    • Characters in a string are added as separate values unless you use the add() function.
      • const letters = new Set('hello'); → { 'h', 'e', 'l', 'o' } 
      • const words = new Set().add('hello').add('world') → {'hello', 'world'}
    • Duplicates are ignored
    • Non-primitive values are considered unique (arrays, objects, etc)
    • Methods
      • setName.size() - finds the number of values in a set
      • setName.has() - checks if a value is in a set
        • Returns a boolean
    • Removing values from sets
      • setName.delete() - removes a value from a set
      • setName.clear() -  removes all values from a set
    • Converting to Arrays
      • Use the spread operator (...) to convert a set to an array
        • const myArray = [...mySet];
      • Can also use the Array.from() method
        • const myArray = Array.from(mySet); 
    • Weak sets
      • Objects are stored in sets as long as the set exists
      • Can cause memory leak - when a program retains references to values that can no longer be accessed in memory. ()dead objects)
        • Wastes resources
      • Weak sets allow “garbage collecting” references to “dead objects”
      • To create - const weak = new WeakSet(); 
      • Only non-primitive data types can be added to these sets
  • Maps
    • A way of keeping key/value pairs
    • Similar to objects with these differences:
      • Objects use strings for key values, keys can use any data type
      • No good way to find number of key/value pairs for objects but maps can use the size property
      • Objects have methods and prototypes to create chain of inheritance but maps are focused on storing and retrieving key/value pairs.
      • Object property values can be accessed directly but maps make you use the get() method.
    • To create a map - const romanNumerals = new Map();
    • To add key/value pair to map - romanNumerals.set(1,'I');
    • “Hash rocket” symbol used to display key/value pairs (=>)
      • {1 => ‘I’}
    • Add multiple values -using the set() method romanNumerals.set(2,'II').set(3,'III').set(4,'IV').set(5,'V') 
    • Methods
      • Use get() to look up a value using the key
      • Use has() to see if a key is in a map  
      • Create a map with multiple values using a nested array as a parameter - const heroes = new Map([ ['Clark Kent','Superman'], ['Bruce Wayne', 'Batman'] ]); 
      • Use the size property to find the number of key/value pairs - 
    • Removing entries from maps
      • mapName.delete() - delete a key/value pair. 
        • Returns a boolean if it worked
      • mapName.clear() - removes all key/value pairs from a map
    • Converting maps to arrays
      • Maps convert to nested arrays of key/value pairs
      • Use spread operator (...)
        • [...romanNumerals] → [ [1,'I'], [2,'II'], [3, 'III'], [4,'IV'], [5,'V'] ] 
      • Or use the Array.from() method
        • Array.from(romanNumerals) → [ [1,'I'], [2,'II'], [3, 'III'], [4,'IV'], [5,'V'] ]
    • Weak maps and sets - good for optimizing memory and avoiding leaks but can’t use all the methods that the regular maps and sets can.
  • Logic - allows you to control the flow of the program
    • if statements
      • Structure - if (condition) { // code to run if condition is true } 
    • else statements - comes after the if statement and gives code to run if the condition is false
      • Structure - if (condition) { // code to run if condition is true } else { // code to run if condition is false }
    • Ternary operator - shorthand for writing if statement (?)
      • Structure - condition ? (//code to run if condition is true) : (//code ➥ to run if condition is false) 
      • Can make code easier to write but harder to read
    • switch statements - use when there are lots of conditions to test
      • Structure - switch(valueToCompare)

case value1: code to execute if the case occurs;

break;

case value1: code to execute if the case occurs;

break;

default: code to execute if nothing above occurs;

Break;

  • Loops - repeat a piece of code over and over
    • while loops - runs a block of code while a condition is true
      • while (condition) { // do something } 
    • Infinite loop -  a situation where the condition of a loop is never met and the loop runs indefinitely
    • do...while loops - similar to while loops but the condition comes after the block of code.
      • do...while loops will always run at least once.
      • Structure - do { do something } while(condition)
    • for loops - commonly used type of loop
      • Structure - for (initialization ; condition ; after) { do something }
        • Initialization - a variable is initialized to use in the loop
        • Condition - needs to be satisfied for the loop to continue
        • After - what happens after each iteration of the loop
      • Nested for loops - a loop inside a loop
        • Inner loop runs completely before the next step of the outer loop runs
      • for loops can be used to iterate over array values
        • Structure - for(let i=0, max=avengers.length; i < max; i++){ console.log(avengers[i]); 
    • For-of loop (for arrays) 
      • Structure - for(const value of avengers){}
        • The variable value needs to be declared as const
    • Looping over sets and maps
      • Use for-of loop to loop over values in a set or key/value pairs in a map
      • Loops go over the values in the order they were added
      • Use keys() method for keys and values() method for values
        • for(const key of romanNumerals.keys()){}
        • for(const value of RomanNumerals.values()){}
      • Use entries() method for both key and value
        • for(const [key,value] of RomanNumerals.entries()) { console.log(`${key} in Roman numerals is ${value}`); } 
      • Can’t loop over values in weak sets and weak maps

Chapter 4: Functions

  • Function - a piece of code referenced by a name
    • Like a self-contained mini program
    • Reduces repetition & makes code easier to follow
    • Functions are first class objects - they behave like all other primitive data types and objects
  • Defining a function
    • Declaration  - creates a function literal
      • Structure -  function functionName() {code to be executed...}
    • Expression - assigns a function to a variable (names or unnamed)
      • Structure 
        • const goodbye = function bye() {code to be executed...}  OR
        • const goodbye = function() {code to be executed...};
          • NOTE: this needs a semicolon at the end b/c it is the end of a statement.
    • Constructors - not recommended because of problems that arise from the method of construction and they have global scope
      • Structure - const hi = new Function('console.log("Hi World!");'); 
  • Invoking a function - to invoke call its name followed by ()
  • DRY code - Don’t Repeat Yourself
    • Every part of a program should only be written once
      • Ex. write a function once the simply invoke it when needed
  • Return values - use the return keyword to define what is to be returned from a function
  • Parameters and Arguments - values given to functions for input
    • Parameter - set when the function is defined (placeholders)
    • Argument - passed to a function when it’s invoked
    • Arguments variable - special variable for functions
      • Returns arguments passed to function as an object
      • Use the rest operator to create an array of arguments
        • To use - place … in front of the last parameter in the declaration
        • Can be used to improve the mean() function
    • Default Parameters - values used by functyions if no arguments are provided
      • Assign default value in function definition - 
        • Ex: function hello(name='World') { console.log(`Hello ${name}!); }
        • Call this function w/o argument = Hello World
        • Call this function with “Universe” argument = Hello Universe
      • Always place default parameters last!
    • Arrow Functions
      • Identified by =>
      • Structure = x=> x*x
      • Always anonymous
      • Parameters before arrow, main body after
      • Assign to variable if you want reference
        • const square = = x=> x*x
      • Advantages
        • Less verbose
        • Single parameters needs no ()
        • Function body doesn’t need to be in block if its only one line
        • Return keyword not required if only one statement
        • They don’t bind their value to of this to the function
      • Best for short anonymous functions
      • Multiple parameters - const add = (x,y) => x + y;
      • No parameters const add = (x,y) => x + y;
    • Function hoisting 
      • Hoisting -  action of moving all variable and function definitions to the top of the current scope
      • Functions created by declaration are automatically hoisted
        • Can be invoked before they are defined
      • Variable hoisting - variables defined with var automatically go to top of scope
        • Can also be invoked before definition but this is problematic because the assignment is NOT hoisted
          • Will be undefined until it is assigned a value
        • Best to use const and let to avoid hoisting issues
      • Functions expressions are  hoisted like variables
        • If declared using var - declaration is hoisted but not the function
    • Callbacks - a function passed to another function as an argument
      • Use the function name as a parameter and call the function from within the other function
    • Array iterators
      • forEach - uses a callback function to pass arguments to as it loops over each value in an array
      • map() - similar to forEach but it replaces each value in the array with what is returned from the callback function
      • reduce() - cumulatively combines results as ti loops and returns a single result
        • Can be used to improve the mean() function
      • filter() - returns a new array of only truthy values the are returned from the callback function
      • Chaining iterators - These iterators can be chained together to complete more complex transformations.
        • Ex- [1,2,3].map( x => x*x ).reduce((acc,x) => acc + x );
          • This uses map() and reduce() to square a set of numbers then add them all up
