Week 10 Notes

Form Validation

Why Validate

  • Get the right data in the right format
  • Protect user’s data (requiring strong passwords)
  • Protect yourself (the developer)

Different types of client side validation

  • Built-in form validation
    • Good performance, not very customizable
    • HTML5 validation: required fields, data type, patterns (regex)
  • JavaScript validation
    • Very customizable but you have to create it

Elements can be styled depending on their validity

  • CSS :valid and :invalid pseudo classes

The Constraint  Validation API

  • Has methods and properties that work on HTML form elements

Using Fetch

The Fetch API allows you to access the http pipeline and request and send information

  • Replaces XMLHttpRequest
  • Really simple to set up:

fetch('http://example.com/movies.json')

  .then(response => response.json())

  .then(data => console.log());

  • The fetch() method can have one optional parameter - the init object
    • The init object allows you to control many different settings
  • Fetch can be used to:
    • Send requests with credentials
    • Upload JSON data
    • Upload files
    • Process text files line by line
  • You can supply your own request object by using the Request() constructor and pass it to the fetch as and argument
  • Headers:
    • Use the Headers() constructor to create a headers object
      • Creates a map of name value pairs
    • Good use for headers is checking valid HTTP header names
  • Response Objects
    • Returned when a fetch promise is resolved
    • Most common: Response.status, Response.statusText, Response.ok
